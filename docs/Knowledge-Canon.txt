# 知識基盤（Knowledge-Canon）：Fisher’s Logic Engine

## 1. プロジェクト正本
### 1.1 プロジェクトの目的
FF14（パッチ7.x以降）の漁師スキル回しを最適化する計算基盤。
サーバー側の「キャスト後の再抽選プロセス」をエミュレートし、実効待機時間と演出硬直を含めた真の効率を算出する。

### 1.2 現在の到達点 (v1.7.0)
- **再抽選モデルの確立**: キャスト後のルアーによるテーブル書き換え論理の定義完了。
- **演出秒数の厳密化**: 振動（！/！！/！！！）に応じたフッキング演出時間の定義（5s/9s/13s）。
- **マスターデータ構造**: スプレッドシート（釣り場、テーブル1、テーブル2）との連携。
