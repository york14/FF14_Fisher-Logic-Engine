# トラブルシューティングログ: アプリ実行時のデータ読み込みエラー

- **発生日時**: 2026-01-27 13:58
- **事象**: `npm run dev` でサーバーは起動したが、ブラウザコンソールおよびターミナルに `Failed to resolve import "./data/logic_master.json"` エラーが発生し、画面が正しく表示されない。
- **エラー内容**: `src/data` ディレクトリが存在せず、参照先のJSONファイルが見つからない。

## 原因
プロジェクト初期化時の `mkdir` コマンド (`mkdir src src\data ...`) が、Windowsコマンドプロンプト上で期待通りに複数の階層を一括作成できていなかった、あるいはコマンド実行の非同期処理中に競合が発生した可能性がある。
結果として `src` フォルダは作成されたが、サブフォルダ `data` が欠落していた。

## 解決策
**ディレクトリの再作成とファイルの配置を行う。**

### 手順
1. `src/data` ディレクトリを作成する。
   ```cmd
   mkdir src\data
   ```
2. 正本データ (`old/logic_master.json`) を所定の位置にコピーする。
   ```cmd
   copy old\logic_master.json src\data\logic_master.json
   ```
3. `npm run dev` を再実行し、エラーが解消されたことを確認する。

## 今後の対策
`mkdir` で階層の深いディレクトリを作成する場合、Windowsでは `mkdir a\b\c` のように一括作成するか、確実に一つずつ作成する（または `-p` 相当の挙動を確認する）。
ファイル操作コマンド後は必ず `dir` 等で結果を確認してから次へ進むこと。
