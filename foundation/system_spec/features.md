# Application Features

FLEが提供する主要な機能・モードのシステム的仕様です。

## 1. モードと機能概要

FLEは画面左側のタブによって以下3つの計算モードを切り替えます。

### 手動設定モード (Manual Mode)
- **概要**: ターゲット魚やルアーのステップ（1回目発見、2回目型確定など）、トレードリリース、撒き餌の有無を単一の「シナリオ（Scenario）」として設定し、その条件に特化した計算を行います。
- **出力**: 各魚のヒット率、サイクル時間、GP収支、ターゲットを1匹釣るための期待時間。

### L戦略評価モード (Strategy Mode)
- **概要**: 用意された2つの戦略セット（セットA、セットB）の設定を比較評価します。
- **出力**: 各戦略における複数シナリオの加重平均結果（平均ヒット率、平均サイクル、総合期待時間）をA・B並べて表示します。

### 総合戦略評価モード (Optimizer Mode)
- **概要**: 制限時間と総GPという制約の中で、「GPを消費するが期待値が高い戦略（赤字戦略）」と「GPが回復する戦略（黒字戦略）」をどのように組み合わせれば最も効率的かを連立方程式で解きます。
- **出力**: セットA・Bそれぞれの最適な使用回数（$n_A, n_B$）、トータルでの期待獲得数、1匹あたりの期待時間。

---

## 2. 変数モード (Variable Mode)
- **概要**: 対象魚の「基本重み（Base Weight）」が不明、またはプレイヤー自身で仮説を立てたい場合に使用するモード。
- **内部挙動**: 内部的にターゲットの確率要因を変数 $p$ として扱い、方程式の係数（フッキング後の時間 $A$ や外道分の加重平均サイクル $B$ 等）を抽出します。（詳細は `simulation_logic.md`）
- **制約**: 釣り場内に隠し魚（Hidden Fish）が存在する場合、システム上 $p$ への分離が難しいため、変数モードのチェックボックスは自動的に無効（Disabled）化されます。

## 3. シェア機能 (Share)
- **概要**: 現在画面に表示されている設定と結果を独自のURLにし、クリップボードにコピーします。
- **内部挙動**: UI状態をJSONオブジェクト化 → JSON文字列化 → Base64エンコードし、`share.html?data=...` のURLを生成。受信側は `share.html` がこれをデコードし、計算を再実行または結果を静的にレンダリングします。
