# System Architecture

FLE v3.x のシステムアーキテクチャの概要です。

## 基本構成
本プロジェクトは、Node.js エコシステムにおける **Vite** を用いてビルドされる **Vanilla JavaScript (ES Modules)** アプリケーションです。
バックエンドサーバー（APIやDB）を持たず、**フロントエンド（ブラウザ上）で完全に完結**して動作します。

## ディレクトリ・モジュール構造

主なソースコードは `src/` 配下に格納されています。

- **`src/main.js`**:
  アプリケーションのエントリポイント。UIとコアロジックの橋渡し、イベントリスナーの登録、状態管理を担う。
- **`src/core/`**:
  UIに依存しない純粋な計算処理モジュール群。
  - `calculator.js`: 釣獲確率、サイクル時間、シナリオごとの期待値計算のメインロジック。
  - `optimizer.js`: GPの消費と回復、GPバランス計算。
  - `scenario.js`: 「発見」「型確定」といったルアーステップの構成ID（`scenarioId`）のパース・生成。
- **`src/ui/`**:
  DOM操作や結果のレンダリングを担当するモジュール群。
  - `controls.js`: セレクトボックスの連動やUI状態の更新。
  - `render.js`: 計算結果のHTML構築、DOMへの適用。
  - `tab_optimizer.js`: 総合戦略評価（オプティマイザ）専用のUI制御・結果描画。
- **`src/data/`**:
  - `logic_master.json`: 釣りの基本データ（釣り場、魚、ルアー確率、重み等）が統合されたJSON。アプリ起動時にインポートし全機能の基盤データとして使用。

## 状態管理とシェア機能
UIの入力状態はJavaScriptのオブジェクトとしてシリアライズされ、Base64エンコード後URLパラメータ（`?data=...`）として持たせることができます。
これにより特定の設定と計算結果の共有（シェア機能）を実現しています (`share.html` / `main.js` 内の `initShareMode`)。
