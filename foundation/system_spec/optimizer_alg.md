# Optimizer Algorithm (総合戦略評価)

`src/ui/tab_optimizer.js` 等で実行される、総合戦略評価のシステム的解法です。
制限時間ならびに総GPの制約の中で、2つの戦略を用いた場合の最適配分数を連立方程式で解きます。

## 1. アルゴリズムの前提
各戦略（戦略セットA、戦略セットB）には、計算ロジックによって以下の値が事前に求まっているものとします。
- $c_A$ : 戦略Aの1サイクルにかかる時間 (Average Cycle)
- $c_B$ : 戦略Bの1サイクルにかかる時間
- $gpNet_A$ : 戦略Aの1サイクルごとのGP収支 (消費量 - 自然回復等)
- $gpNet_B$ : 戦略Bの1サイクルごとのGP収支

前提として、戦略セットAは「GP赤字（期待値は高いがGPが減っていく）」、戦略セットBは「GP黒字（期待値はさほどでもないがGPが増える）」の状態を想定しています。

## 2. 条件変数の定義
UI側でユーザーが指定する制約条件：
- **T** : 制限時間（秒）
- **GP0** : 初期GP
- **S** : サリャクの恩寵 使用見込み回数
  - サリャク1回につき `150` のGPを追加リソースとして換算。
- **TotalGP** : $GP0 + (S \times 150)$

## 3. 連立方程式の構築と解法
戦略Aの使用回数を $n_A$、戦略Bの使用回数を $n_B$ とし、以下を同時に満たす状態を連立方程式として解きます。

1. **時間制約**: $$ n_A \times c_A + n_B \times c_B = T $$
2. **GP制約**: $$ n_A \times gpNet_A + n_B \times gpNet_B = -TotalGP $$

上記を行列形式・クラメルの公式等に当てはめ、行列式 (Determinant): 
$$ det = c_A \times gpNet_B - c_B \times gpNet_A $$
から $n_A, n_B$ を算出します。

## 4. 計算例外と無効判定
システムでは、連立方程式を解いた後、以下のチェックを行います。
- **解無し（$det \approx 0$）**: 2つの戦略のGP効率・時間効率が完全に平行な場合。
- **回数がマイナス（$n_A < 0$ または $n_B < 0$）**: 与えられた時間やGPに余裕がありすぎて、片方の戦略（特に黒字戦略）を一度も使う必要がない場合など。この場合、最適解としては不成立（Invalid）としてUI上に警告を表示します。

## 5. 変数モードとの統合
オプティマイザ機能を変数モードで実行した場合、方程式の係数（サイクル時間 $c_A, c_B$ 等）が $p$（仮説重み確率）に依存した形になります。
そのため、システムとしては $p$ を事前に定義したサンプルポイント群（例: 1%, 5%, 50%, 95%など）のそれぞれについて上記連立方程式を解き、結果の一覧を表形式でレンダリングするアプローチを取っています。
