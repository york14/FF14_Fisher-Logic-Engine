# Game Mechanics

FF14におけるフッキング〜釣り上げのサイクル時間モデルについて、実装ベースの定義です。

## サイクル時間の構成要素
1回のキャスティングアクションにおける全体の所要時間は、以下のフェーズの合計として定義されます。

### 1. キャスト (Cast)
キャスティングに伴う基本動作時間 `$D_CAST$`（約1.0秒）。

### 2. ルアーアクション (Lure Action)
ルアーを使用する場合、1ステップにつき `$D_LURE$`（約2.5秒）を消費。
さらにルアー使用時には終了処理ブロック時間 `$D_BLK$`（約2.5秒）が加算される。
※ルアーを使用しない場合、このフェーズの時間は0。

### 3. 待ち時間 (Wait Time)
魚が食いつく（Bite）までの時間。
魚ごとに設定された `bite_time_min` 〜 `bite_time_max` と、現在のルアーアクション消費時間を比較し、システムは積分的なアプローチを用いて「平均待ち時間（Wait Time Avg）」を算出します。

### 4. フッキング＆釣り上げ (Hooking & Catch)
ヒットした魚に対する釣り上げ所要時間。
- ターゲット魚、または「外道も全て釣る」オプションONの場合: 魚ごとの `hook_time` を消費。
- 不要な外道（フッキングしない / 途中で竿上げ等）の場合: リセット時間 `$D_REST$`（約2.0秒）を消費。

### 5. サイクル終了と待機 (Post-Cast / Pre-Cast)
一部スキル（撒き餌等）使用時の追加時間消費（後述）。

## 竿上げ（未発見即中断）
シナリオ構築時、「発見（Discovery）」が発生しなかった場合に即座に釣りを中断する「未発見即竿上げ（Quit）」という分岐があります。
この場合、Wait TimeとHookingの代わりに `$D_REST$` モーションを用いた短縮サイクルが成立し、サイクルの平均所要時間が大幅に減少します。
