# ドメイン知識: メカニクスと属性

本書は、FLEシミュレータで使用されるFF14の「釣り」ドメインの基本概念、ルール、用語を定義する。

## 1. 釣りサイクル (Fishing Cycle)
「釣りサイクル」はシミュレーションの最小単位であり、キャスト開始からそのキャストの結果が確定するまでの一連の流れを指す。

### 1.1 サイクルの構成ステップ
1.  **Pre-Cast (準備)**: 任意の準備動作 (例: 撒き餌の使用)。
2.  **Casting (キャスト)**: 竿を振るアニメーション。
3.  **Lure Application (ルアー適用)** (FLE固有概念):
    *   特定のルアー戦略が有効な場合、水中で竿を操作する（振る）時間。
    *   これにより、開始直後の「魚がかからない時間（拘束時間）」が増加する。
4.  **Wait (待機)**: 魚がバイトするのを待つ時間。
5.  **Bite (アタリ)**: サーバーから魚の接触が通知される (振動/SE)。
6.  **Resolution (解決)**:
    *   **Hook (釣り上げ)**: フッキング成功時。魚を獲得する。
    *   **Rest/Quit (竿上げ)**: 魚を釣らずに中断する、またはアタリが来る前に中断する。

## 2. 魚の属性

### 2.1 型 (Jaws / Fish Size Type)
ルアー (Ambitious/Modest) との相互作用を決定する。
*   **小型 (Small Jaws)**: モデストルアー (Modest Lure) に反応。
*   **大型 (Large Jaws)**: アンビシャスルアー (Ambitious Lure) に反応。

### 2.2 メカニクス属性
*   **Hook Time (釣り上げ時間)**: その魚を釣り上げる拘束時間。ゲームデータ固定値。
*   **Bite Time (待機時間)**: キャスト後、その魚にアタリが発生する可能性がある時間範囲 (Min-Max)。
*   **Vibration (振動)**: ユーザーに表示される引きの強さ ( !, !!, !!! )。

### 2.3 特殊分類
*   **Hidden Fish (隠し魚)**: 通常の抽選テーブルには出現しない。「発見 (Discovery)」イベントが発生した時のみ出現する。
*   **Rare Species (希少種/ヌシ)**: 特定の条件下でのみ釣れる高価値のターゲット。

## 3. ルアーメカニクス (Ambitious/Modest)
深海釣りや特殊ゾーンなど、特定のコンテンツで用いられるギミック。

### 3.1 効果
ルアーを使用すると**スタック** (1, 2, 3回) を消費し、2つの効果を発生させる:
1.  **Weight Modifier (重み補正)**: 一致する型の魚の重みを増加させる。
2.  **Status Induction (状態付与)**:
    *   **Discovery (発見)**: 隠し魚を出現させる。
    *   **Guarantee (型確定)**: 次のバイトの「型」を固定する (例: 次は必ず大型)。

### 3.2 競合ルール
*   **Guarantee Override (確定優先)**: 型確定が発生した場合、**反対の型**を持つ魚の重みは強制的に 0 になる。
*   **Multi-Step (多段判定)**: ルアーアクションは順次行われる (例: 3回振り)。状態付与の抽選はステップごとに独立して行われる。

## 4. スキル

*   **Surface Slap (トレードリリース)**:
    *   直前に釣った魚(指定魚)を抽選プールから除外する (Weight = 0)。
    *   コスト: 200 GP。
*   **Chum (撒き餌)**:
    *   全魚種の待機時間を短縮する (本シミュレータでは基本的に x0.5)。
    *   コスト: 145 GP。
